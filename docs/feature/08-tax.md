# Feature Spec — Tax (หน้าภาษี)

สรุปฟีเจอร์และ UX จาก research

---

## Research & UX Analysis (จาก GPT Research MCP)

**ที่มา:** เรียก MCP tool `research` (server: project-0-account-stock-fe-mcp) โดยส่งคำอธิบายหน้าภาษีและคำขอให้วิเคราะห์ว่าผู้ใช้ร้านขายของทั่วไปอยากเห็นอะไร (เช่น คุณอยู่ในฐานไหนจากยอด เสียภาษีกี่บาท) และควร ADD / REMOVE / REORGANIZE อะไรเพื่อให้ user ทั่วไปเข้าใจ

---

### สรุปจาก GPT Research MCP

**ADD**
- แสดงสถานะภาษีชัดเจนจากรายได้: อยู่ในช่วงอัตราไหน (tax bracket)
- สรุปตัวเลขหลัก: รายได้รวม, เงินได้สุทธิ, ภาษีที่ต้องจ่ายหรือได้คืน
- สถานะภาพ (สีแดง/เขียว): ต้องจ่ายเพิ่ม vs ได้คืน
- คำแนะนำลดภาษีแบบเข้าใจง่าย สำหรับ seller/SME (เช่น หักค่าใช้จ่าย, ลดหย่อน)
- รายการลดหย่อนที่พบบ่อย + อธิบายแบบภาษาชาวบ้าน
- FAQ หรือความช่วยเหลือสำหรับคำถามที่ user มักถาม
- ปุ่มคัดลอก/ส่งออกสรุป

**REMOVE**
- ศัพท์บัญชี/กฎหมายที่ซับซ้อนหรือไม่จำเป็นต่อการตัดสินใจทันที
- ฟิลด์ที่ร้านขายของทั่วไปไม่ใช้บ่อย
- layout แออัดหรือรายละเอียดขั้นบันไดที่ overwhelming

**REORGANIZE**
- วางผลสรุป (ภาษีที่ต้องจ่าย/ได้คืน) ไว้บนหรือตำแหน่งเด่น
- จัดกลุ่ม input/output ตามลำดับ: รายได้ → เงินได้สุทธิ → คำนวณภาษี → tips
- ใช้ส่วนย่อได้ (collapsible) หรือแท็บแยกรายละเอียดออกจากมุมมองสรุป
- เน้นความอ่านง่าย: whitespace และ icon

**Trade-offs**
- Tips และคำอธิบายช่วยความเข้าใจ แต่จัดไม่ดีจะรก
- ภาษาง่ายช่วย non-accountant แต่อาจ oversimplify
- รายละเอียดน้อยลงช่วยมือใหม่ แต่ power user อาจอยากเห็นเต็ม

## การดำเนินการ (ที่ implement แล้ว)

| หมวด | การดำเนินการ |
|------|----------------|
| **Hero** | บล็อก "คุณอยู่ในฐานไหน" — แสดงช่วงอัตราภาษี (bracket) + ตัวเลขใหญ่ "เสีย X บาท" หรือ "ได้คืน X บาท" (สีแดง/เขียว) |
| **Summary** | การ์ดสรุป 4 ตัว: รายได้รวม, หักลดหย่อนรวม, เงินได้สุทธิ, ภาษีที่ต้องจ่าย/ได้คืน |
| **Export** | ปุ่ม "คัดลอกสรุป" — คัดลอกข้อความสรุปไป clipboard |
| **Input** | จัดกลุ่ม: รายได้ → วิธีหักค่าใช้จ่าย → ค่าลดหย่อน (ภาษาง่าย) |
| **Result** | ผลภาษีชัด: ภาษีที่ต้องจ่าย, หัก ณ ที่จ่าย, สุทธิที่ต้องจ่าย/ได้คืน (สีแดง/เขียว) |
| **Tips** | บล็อก "วิธีลดภาษี" — SSF/RMF, ประกันชีวิต แบบ layman + ตัวเลขประมาณการ |
| **รายละเอียด** | รายละเอียดขั้นบันไดและตารางอัตราเต็ม ใส่ใน `<details>` (collapsible) เพื่อไม่รก |
